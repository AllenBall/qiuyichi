# 推演方法（每场推演前必读）

> 脱敏自企业级系统的反应生成与评价逻辑。核心思想一句话：**行为和心理分开记**——人嘴上说不要，手上点进去，两层都要抓到，缺一层都是失真。

## 第 0 步，规划本场评估表

每场推演开始前，先按方案类型定一张评估表（**本场所有画像共用同一张表**，否则没有可比性）：

- **行为清单**（4–6 项，每项只答 是/否）。按方案类型选：
  - 内容类：看完 / 点进去 / 收藏 / 转发 / 关注
  - 定价类：点进去 / 看价格详情 / 试用或加购 / 付费 / 划走
  - 功能类：注意到 / 用了一次 / 再次使用 / 推荐给别人
  - **标注依赖关系**：如「付费 ← 需先 点进去」（漏斗关系，第 2 步要用）
  - 行为分两类：**主漏斗行为**（有依赖链、参与第 2 步强制自检）和**支线行为**（如「去比价」「截图发群」，独立发生、不进漏斗链）。规划时明确标出哪些是支线，避免自检时把支线误当前置
- **心理维度**（3–5 项，0–100）。默认三件套：想不想点 / 看不看得懂 / 会不会推荐；按场景增补——定价场景加「觉得值不值」，品牌场景加「信任感」，高频功能加「烦不烦」。
- **北极星行为**：本场最重要的一个行为（如「付费」）。报告排序与多方案对比以它为准。

评估表用一句话念给用户过目，没意见就开跑。底线：行为 ≥2、心理 ≥2；凑不齐就回退默认（三件套 + 点进去/划走）。

## 第 1 步，逐人生成双层反应

一次只入戏一个画像（小传 + 个人记忆 + 全部世界记忆读入），按固定顺序输出：

1. **心理独白**（两三句：这个人看到方案的瞬间，脑子里闪过什么。**先写独白再打分**——分数必须从独白里长出来，不许凭空给）
2. **心理打分**（按评估表逐维 0–100）
3. **行为判定**（按评估表逐项 是/否。允许心口不一：嘴上嫌弃、手滑点进去是真实人性，但 reasoning 里要交代为什么）
4. **一句原话**（角色口气）
5. **改进建议**（站他的立场，一条就够，没有写「无」）

## 第 2 步，漏斗一致性自检

逐人检查行为依赖（只查主漏斗行为，支线行为不参与强制）：**前置行为为「否」的，后续行为强制改「否」**（没点进去就谈不上付费）。只改行为，**心理分和独白保留**——「心理上想付钱但没点进去」本身就是重要信号：这是入口问题，不是价格问题。

## 第 3 步，汇总

- **行为漏斗**：N 人看到 → x 人点进去 → … → y 人完成北极星行为。**漏斗断在哪一级，问题就在哪一级**——这一行是整份报告里最值钱的一行
- **心理均分**，以及每个维度的最高分者与最低分者（分歧的线索）
- **推演纪实**：一到两段叙事，把这场推演讲给没在场的人听（谁先动、谁划走、转折点、关键原话织进去）。独白与原话的心口不一处，往往就是纪实里最值得点出的洞察
- **分主题反馈归组**：把所有人的反馈按主题聚类（价格 / 第一眼 / 信任感 / 易用性…主题从反应里自然长出，不硬套），每主题标提及人数，每条注明出处画像——这是把「一堆人说话」变成「几件事有多少人在意」的关键一步
- 共识 / 分歧（谁、为什么、基于他哪条设定）
- **多候选方案对比**：按北极星行为完成人数排序，平票看心理均分

## 存档结构（写入推演记录）

每人一行：姓名 ｜ 行为（✓/✗ 串，按评估表顺序）｜ 心理分（维度=分…）｜ 原话 ｜ 建议
漏斗一行：看到 N → 点进去 x → …（北极星）y
本场评估表随记录存档（行为清单 / 心理维度 / 北极星 / 依赖关系）。
