# 画像方法（建画像时必读）

> 本方法脱敏自一套企业级用户仿真系统。画像质量决定推演上限——纸片人只能给出纸片反应。

## 生成三步法

**第一步，定骨架（结构化字段 + 池组合）。** 先把这个人的字段填出来（见下方字段表），并按「五维度池」规则领取差异化组合。骨架决定这个人「是谁」，防止写小传时凭感觉漂移。

**第二步，写小传（四层结构）。** 400–600 字第一人称自述，按顺序写四层：

1. **我是谁**——基本盘：年龄、身份、生计、家庭
2. **我的性格**——脾气、决策习惯、钱观（他怎么做决定，比他是谁更重要）
3. **我和这类产品的关系**——怎么发现的、用多深、为什么用、在哪不爽
4. **独属于我的细节**——两三件具体小事（周三晚上开黑、错题本每周日过一遍）

两条铁纪律：
- **产品相关内容不超过全文 1/3**。一个满脑子都是你产品的用户是纸板做的——真人的生活里，你的产品只占一个角落，小传的大部分篇幅必须是「这个人本身」。
- **禁止数据术语**。小传里不许出现「转化率」「用户群体」「活跃度」这类词——人不会这样介绍自己。

**第三步，真话锚（用户提供）。** 从评论区、客服记录、群聊里找一句这类人真说过的话。拿不出就标「待补」，**绝不杜撰**。

## 结构化字段表（每个画像必填）

| 字段 | 说明 |
|------|------|
| 姓名 / 年龄 / 性别 | 给 AI 一个可入戏的人 |
| 城市层级 | 一线 / 二三线 / 县城乡镇（消费语境差异极大） |
| 职业身份 | 决定作息、语言习惯和决策场景 |
| 预算带 | 不写具体收入也行，写「什么钱舍得、什么钱心疼」 |
| 性格类型 | 一句话或 MBTI 均可，与小传第 2 层呼应 |
| 付费敏感度 | 高 / 中 / 低 |
| 设备与渠道 | 安卓/苹果、中端/旗舰——影响体验预期 |
| 使用深度 | 尝过一次 → 重度依赖，五档 |
| 使用动机 | 为什么用这类产品 |
| 所属产品 | 推演前按此匹配，防串戏 |

## 五维度池（差异化的引擎）

同类用户会长得一模一样——都是「宝妈」，凭什么一个爱囤货一个只买急用？解法是从五个池子里给每人发不同的组合：

| 池 | 取值示例（可按你的业务替换扩充） |
|----|------|
| ① 发现渠道 | 朋友推荐 / 刷到广告 / 自己搜的 / 同事在用 / 博主安利 / 被拉进群 |
| ② 使用深度 | 只用过一次 / 偶尔想起 / 固定场景用 / 每天都用 / 重度并爱折腾 |
| ③ 使用动机 | 燃眉之急 / 提效省时 / 跟风尝鲜 / 给家人用 / 工作要求 / 省钱替代 |
| ④ 生活情境 | 单身租房 / 带娃家庭 / 照顾老人 / 异地通勤 / 自由职业在家 / 学生宿舍 |
| ⑤ 性格特质 | 抠门精算 / 急性子 / 怀疑一切 / 好好先生但从不付费 / 收藏不行动 / 较真爱写差评 |

**确定性组合规则**：建第 k 个画像时，第 j 个池取第 `((k + j) mod 池长度) + 1` 个值。**k 是这个画像在画像库中的全局序号**（沿用库内既有编号顺延，示例画像不计入）——跨批次也按同一序列取，保证整库不重样、可复现。这样做的原因：随机组合每次重跑长出不同的人，出了问题没法复现；确定性组合让同一批画像可以被反复审计，也天然保证了批内不重样。组合领到后，把五个取值**写进小传对应的层里**（渠道和深度进第 3 层，情境进第 1 层，性格进第 2 层），并在画像的「池组合」栏留档。

## 质量自检（写完每个画像过一遍）

- 遮住名字读小传，这个人和库里其他人**会在同一个方案上吵起来**吗？不会就回炉
- 小传里有没有至少两件「只属于他」的具体小事？
- 产品相关篇幅超过 1/3 了吗？超了就砍
- 有没有数据术语混进去？
