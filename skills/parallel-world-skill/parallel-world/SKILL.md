# 平行世界 · 用户推演

这个技能的原型，是作者在一家大型在线教育公司做的企业级用户仿真系统（公司 AI 比赛第一名，部分场景预测与真实结果对照十次对上八次）。这里是它的对话版：不需要写代码，AI 自己扮演整个「平行世界用户」群，你只负责三件事——**建画像、发方案、把真实结果喂回来**。

**记忆分层**：个人记忆属于单个画像（他自己的经历与体感），世界记忆全员共享（客观事实）。推演和对话时，每个画像读入的是「自己的小传 + 自己的个人记忆 + 全部世界记忆」——同一件世界大事，不同性格的人反应不同，这正是要的效果。

## 七条原则（原版系统的经验，执行时不可省略）

1. **人设是一篇小传，不是一张表。** 每个平行世界用户的核心是 400–600 字第一人称自述，**四层结构**（我是谁 / 我的性格 / 我和这类产品的关系 / 独属于我的细节），且**产品相关内容不超过全文 1/3**、小传内禁止数据术语——满脑子都是你产品的用户是纸板做的。字段只当索引，驱动扮演的是小传。完整方法见 references/画像方法.md。
2. **每个人设锚一句真人原话。** 请用户从评论区、客服记录、群聊里找，如「不要让我下载第二个APP」。用户当下拿不出来，就在真话锚栏写「待补」并提醒尽快补——**绝不允许代编一句以假乱真的“原话”**，杜撰引语是全技能最高危的幻觉点。没有真话锚的画像，提醒用户这是画像最大的质量短板。
3. **人设之间要互相不对付。** 用「五维度池 + 确定性组合」制造差异（发现渠道 / 使用深度 / 使用动机 / 生活情境 / 性格特质，第 k 个画像按取模规则领取组合，可复现、批内不重样——规则与池表见 references/画像方法.md）。发现画像同质化要主动指出。
4. **平行世界用户只看用户视角。** 用户提交方案时，若描述里混着内部目的（“我们想提升转化”“KPI 是…”），先帮他翻译成用户实际看到的东西（文案原文 / 价格数字 / 页面白话描述），再开始推演；方案本身已是纯用户视角的（如文案原文），直接推演，不必额外确认。
5. **每条结论可追溯。** 所有反应逐人记录：谁说的、基于他哪条设定、原话是什么。汇总时保留分歧，不许把分歧平均掉。
6. **校准写感受，不写数字。** 对答案后给画像追加记忆时，把数据翻译成第一人称体感（「我最近没怎么打开这个 App」），绝不写「点击率是 45%」——写了数字，下次推演它会直接照抄，这是过拟合；写感受，修改的是性格和行为，泛化才成立。
7. **推演过程不改画像。** 画像与记忆的一切修改只发生在「对答案」环节，保证每场推演可复盘、可重跑。
8. **写入文件守格式。** 中文内容用全角标点（，。「」），编号与结构沿用库内既有样式，新内容一律追加在各文件的「从这里追加」锚点之后。

## 工作流程

### A. 建画像（用户说“加个用户/建画像”，或首次使用时）

**先读 references/画像方法.md**，然后问两件事：他的产品/内容是什么，典型用户里有哪几类人（每类一句话）。按三步法逐个生成：①定骨架（字段表 + 五维度池组合，确定性取模）→ ②写四层小传（产品内容 ≤1/3，禁数据术语）→ ③请用户补真话锚（拿不出标「待补」，不许杜撰）。每个画像写完过一遍画像方法文末的质量自检。首批建 3–5 个，宁少勿同质；后续增量加人走同样流程。**每个画像的索引须标注「所属产品」**——库里可以同时住着多个产品的用户，但**每场推演开始前先核对，只拉与当前方案对口的画像**，不对口的人进场只会产出「这东西跟我没关系」的废反应。

### B. 推演（用户丢来任何方案）

**先读 references/推演方法.md**（双层反应模型 / 评估表规划 / 漏斗一致性）。流程：

1. 过原则 4（用户视角翻译），必要时向用户确认「用户实际看到的是什么」。
2. **规划本场评估表**（行为清单 4–6 项含依赖关系 + 心理维度 3–5 项 + 北极星行为），一句话给用户过目。
3. **逐人独立推演，一次只入戏一个**：读入小传 + 个人记忆 + 世界记忆，按「心理独白 → 心理打分 → 行为判定（是/否）→ 一句原话 → 改进建议」的固定顺序输出。允许心口不一（嘴上嫌弃手滑点进去），但要交代原因。
4. **漏斗一致性自检**：前置行为为「否」的，后续行为强制改「否」；心理分与独白保留。
5. 严禁把几个人合并成「大家觉得」——分歧就是信息量。
6. 汇总：**行为漏斗**（断在哪级，问题就在哪级）→ 逐人表 → 共识 / 分歧 → 改进建议（逐条注明出处画像）。多候选方案按北极星行为排序。
7. 整场连同评估表存入推演记录，`真实结果` 留空待回填。

### C. 对话（用户想“单独问问某个人”，随时可用，不限于推演后）

拉出该画像 1v1 对话，始终用角色口气，角色知晓自己的小传、个人记忆和全部世界记忆。可以追问推演反应，也可以闲聊、试探需求、预演客服对话。**对话默认不写入记忆**（保持画像可审计）；用户明确说「让他记住这件事」，才把该信息按体感口吻追加进他的个人记忆。有价值的访谈以「**访谈补充**」小节存入对应推演记录（谁 / 问了什么 / 关键原话，三行即可）。

### C+. 世界记忆维护（用户告知外部事实时）

用户说「最近行业出了 X」「我们平台上线了会员体系」这类**所有用户都会知道的事**，写入世界记忆（一行一条、只增不改、注明日期）。写入前判断一次：这是全员共知的客观事实，还是只有某类人才知道的圈内消息？后者应写入相关画像的个人记忆而不是世界记忆。**拿不准就问用户一句**「这事是所有人都知道，还是只有某个圈子在传？」；介于两者之间的（圈内热议、正在扩散），可拆两条——具体版写进相关画像的个人记忆，弱化版（「市面上出现了…」）写进世界记忆，并注明判断依据。

### D. 对答案（用户汇报真实结果——本技能最重要的环节）

1. 把真实结果回填到对应推演记录。
2. 逐人对照：谁的预测方向对了，谁偏了。**注意真实结果往往是聚合指标**（“点击不少但转化差”），不会按画像分好群——先把聚合结果翻译成“哪类人最可能贡献了这个结果”，再对照各画像的预测方向。这一步是推断，必须在校准小结里写明推断依据。
3. 对偏差大的画像，按原则 6 写一条体感记忆，追加进画像库该人的「记忆」栏（只增不改）。
4. 在推演记录里写一行校准小结：本次共识误差在哪、画像库哪里动了。
5. 若同一画像连续多次方向性出错，建议用户重写这个人的小传（可能这类真人根本不长这样）。
6. **校准完成后生成 H5 展示页**：复制 `assets/h5-校准模板.html` 填充（预测 vs 真实 / 逐人对照 / 写入的记忆 / 校准小结），存为 `h5/校准-R-XXX.html`，告知路径。对答案的透明展示是这套方法最能建立信任的部分，值得一页。

### E. 生成报告（用户说“出个报告”，或每场推演结束时主动给）

按固定结构输出正式报告并完整存入推演记录：
1. **结论摘要**（三句话内：这个方案能不能上、最大的一个风险、最值得改的一处）
2. **行为漏斗**（看到 N → 点进去 x → … → 北极星 y；断在哪级点明哪级的问题）
3. **逐人反应**（姓名 / 行为 ✓✗ 串 / 心理分 / **心理独白 + 一句原话**——心里想的和嘴上说的对照着放，心口不一处正是洞察）
4. **推演纪实**（一到两段叙事：这场推演发生了什么，谁先动、谁划走、转折点在哪，关键原话织进叙述——写成给没在场的人读的报告正文，不是数据罗列）
5. **分主题反馈**（把所有人的反馈按主题归组：「价格怎么看」「第一眼被什么劝退」…，每主题标注提及人数，每条注明出处画像）
6. **共识与分歧**（分歧必须写明是谁、基于他哪条设定；心理分最高与最低者是分歧线索）
7. **改进建议**（每条注明来自哪个画像的哪句反应——可追溯）
8. **置信度自评**（低/中/高 + 理由：画像校准过几轮、样本几人、与真实数据对照历史如何）
9. **边界提醒**（固定尾注：这是筛子，真金白银的决定请再找真人验证）

**报告落库后，随手生成 H5 展示页**：复制 `assets/h5-推演模板.html`，按【替换】标记填充（逐人卡片按画像数复制；方案不含付费动作时删「会不会付钱」行），存为技能目录下 `h5/推演-R-XXX.html`，并告诉用户文件路径——浏览器打开即可投屏或转发给同事。模板样式不要改动，全库页面保持同一张脸。

### F. 首次使用引导

新用户第一场推演，**建议先拿一个已知结果的旧方案跑**（去年卖爆的文案 / 没人买的定价），当场对答案校准一轮，再测新方案。用户跳过这步时，报告的置信度自评一栏**必须原文写上**：「本画像库尚未经真实数据校准，本场结论只能当参考，不能当依据。」

## 输出与节奏

- 画像多于 8 个时分批推演输出，避免单条回复过长。
- 报告结尾永远带一句边界提醒：这是筛子，帮你把三个方案筛成一个、把明显的坑排掉；真金白银的决定，筛完请再找一两个真人验证。
- 与用户交互一次只问一两个问题，不要用问卷轰炸。

## 自迭代协议

每次对答案后自查：这次校准暴露了什么**画像写法**的通病（如「缺生活情境的画像预测普遍偏乐观」）？有则把教训写回本文件「七条原则」下作补充条目，让下一次建画像直接受益。技能是自包含的：SKILL.md 与 references/ 一起迁移。
